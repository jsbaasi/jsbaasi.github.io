I use lazyvim.
- `:Noice` to see full error message bubbles
- `:messages` to see lazyvim editor messages (not sure)
- if there's lsp errors that are a bit confusing, maybe the working directory for the lsp is not set correctly. use  `:cd` to correct this
- i have `:ProjectAdd` as a command to register the current directory in my projects list
- `nvim/lua/plugins/python-lsp-settings.lua` is where i can configure to python stuff
- `:set relativenumber`
- gg=G and then CTRL+O to format the entire file
- `:sy` and then e.g `sh` for bash syntax colouring
# How to:
## Find and replace
`:%s/<start>/<end>/g` g for the whole file, not sure what s does exactly
## Clear all highlighting
`CTRL + L`, look into `:h keepp` and apparently `:noh`
## Go back to where your cursor was
`CTRL + O`
## Move around windows
`CTRL + W` and then `CTRL + hjkl`
## Search across all files
`:grep 'pattern'` searches current directory automatically though if different then specify it at the end. `:cw` or `:copen` to see the command output or press `Enter` when it first shows
